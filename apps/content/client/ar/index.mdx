## نظرة عامة

يربط NewYolk بين الزراعة التقليدية والتكنولوجيا المالية الحديثة، حيث يقدم نموذج استثمار فريد يمكن للعملاء من خلاله شراء الدجاج وكسب الإيرادات من مبيعات صفار البيض. مع العمليات الفعلية في رومانيا والعراق، توفر المنصة دعماً زراعياً حقيقياً للاستثمارات الرقمية.

تم بناء NewYolk بهيكلية monorepo حديثة، وتقدم حلاً كاملاً يشمل العمليات الداخلية ومشاركة العملاء والتجارب المحمولة.

## هيكلية Monorepo

المنصة منظمة كـ monorepo مع أربعة تطبيقات رئيسية، كل منها يخدم وظائف تجارية محددة:

### `/apps/crm` - نظام العمليات الداخلية
**التكنولوجيا**: Next.js مع TypeScript  
**الهدف**: إدارة عمليات الموظفين والأعمال

#### وحدة المبيعات
- **إدارة العملاء المحتملين**: تتبع شامل للعملاء المحتملين وخط أنابيب التحويل
- **لوحة المبيعات**: مقاييس الأداء في الوقت الفعلي والتحليلات
- **إدخال العملاء**: تسجيل مبسط وتفعيل الحساب
- **تتبع العمولات**: إدارة أداء المبيعات والتعويضات الآلية

#### وحدة الاحتفاظ
- **إدارة المحافظ**: إشراف على حسابات العملاء والاستثمارات
- **تحليلات الاحتفاظ**: رؤى مشاركة العملاء والربحية
- **أدوات التجديد**: حملات البيع الإضافي وإعادة الاستثمار الآلية
- **إدارة الدعم**: نظام تذاكر متكامل وخدمة العملاء

### `/apps/marketing` - موقع العملاء
**التكنولوجيا**: Next.js مع TypeScript  
**الهدف**: المنصة الأساسية المواجهة للعملاء للاستثمار وإدارة الحساب

#### الميزات الأساسية
- **المصادقة الآمنة**: نظام تسجيل دخول قائم على JWT/OAuth
- **إدارة الاستثمارات**: شراء الدجاج، تتبع الأداء، عرض المحفظة
- **العمليات المالية**: إدارة المحفظة، السحوبات، تاريخ المعاملات
- **التواصل**: الإشعارات والتحديثات في الوقت الفعلي
- **التوطين**: دعم متعدد اللغات (الأسواق الرومانية والعراقية)
- **صفحات الهبوط**: صفحات مُحسَّنة للتسويق والتحويل

### `/apps/expo` - التطبيق المحمول
**التكنولوجيا**: Expo (React Native)  
**الهدف**: تطبيق محمول متعدد المنصات لـ iOS و Android

#### القدرات الرئيسية
- **لوحة الاستثمار**: تتبع المحفظة والأرباح في الوقت الفعلي
- **الإشعارات الفورية**: تنبيهات فورية للمبيعات والمدفوعات والفرص
- **التجارة المحمولة**: تدفقات شراء واستثمار مبسطة
- **المدفوعات الآمنة**: دعم بوابة الدفع المتكاملة
- **الدعم دون اتصال**: الميزات الأساسية متاحة بدون إنترنت
- **الميزات الأصلية**: الكاميرا، المصادقة البيومترية، إشعارات الجهاز

### `/apps/docs` - موقع التوثيق
**التكنولوجيا**: Next.js مع MDX  
**الهدف**: توثيق شامل وموارد المطورين

#### الميزات
- **توثيق API**: مراجع كاملة لـ REST/GraphQL API
- **أدلة المطورين**: دروس التكامل وأفضل الممارسات
- **توثيق المستخدم**: أدلة استخدام المنصة والأسئلة الشائعة
- **مكتبة المكونات**: توثيق مكونات UI المشتركة
- **مستندات الهيكلية**: تصميم النظام والمواصفات التقنية

## البدء

### المتطلبات الأساسية
- Node.js 18+ 
- pnpm (مدير الحزم المُوصى به)
- Git
- Expo CLI (لتطوير التطبيقات المحمولة)

### إعداد Monorepo

1. **استنساخ المستودع**
   ```bash
   npx degit daltonmenezes/newyolk your_project_name
   cd your_project_name
   ```

2. **تثبيت التبعيات**
   ```bash
   pnpm install
   ```

3. **بدء جميع التطبيقات**
   ```bash
   # بدء جميع التطبيقات في وقت واحد
   pnpm dev

   # أو بدء التطبيقات الفردية
   pnpm dev:crm        # نظام CRM
   pnpm dev:marketing  # موقع العملاء
   pnpm dev:expo       # التطبيق المحمول
   pnpm dev:docs       # موقع التوثيق
   ```

4. **الوصول للتطبيقات**
   - CRM: [http://localhost:3001](http://localhost:3001)
   - التسويق: [http://localhost:3000](http://localhost:3000)
   - المستندات: [http://localhost:3002](http://localhost:3002)
   - Expo: محاكي الهاتف المحمول أو تطبيق Expo Go

## مجموعة التقنيات

### التقنيات المشتركة
- **مدير الحزم**: pnpm مع دعم workspace
- **TypeScript**: فحص نوع صارم عبر جميع التطبيقات
- **Tailwind CSS**: نظام تصميم متسق
- **مكونات UI**: مكتبة مكونات shadcn/ui
- **المصادقة**: تطبيق JWT/OAuth
- **قاعدة البيانات**: طبقة قاعدة بيانات مشتركة
- **API**: نقاط نهاية GraphQL/REST API

### التقنيات الخاصة بالتطبيق
- **CRM والتسويق**: Next.js 14، React Server Components
- **Expo**: React Native، Expo SDK، EAS Build
- **المستندات**: Next.js مع MDX، Nextra أو مشابه
- **المحمول**: وحدات أصلية للكاميرا والمدفوعات والبيومترية

## نموذج العمل

يعمل NewYolk على نموذج استثمار زراعي شفاف:

1. **استثمار العميل**: شراء الدجاج الرقمي من خلال المنصة
2. **العمليات الفعلية**: المزارع الحقيقية في رومانيا والعراق تدير الماشية
3. **توليد الإيرادات**: مبيعات صفار البيض تولد عوائد للمستثمرين
4. **توزيع الأرباح**: مدفوعات منتظمة بناءً على مقاييس الأداء
5. **خيارات إعادة الاستثمار**: توسيع الاستثمارات من خلال أدوات المنصة

## التركيز على السوق

- **الأسواق الأساسية**: رومانيا والعراق
- **الجمهور المستهدف**: المستثمرون المهرة رقمياً الذين يبحثون عن الأصول البديلة
- **قابلية التطوير**: بنية تحتية مصممة للتوسع إلى أسواق إضافية

## الفوائد الرئيسية

- **الشفافية**: تتبع في الوقت الفعلي للاستثمارات والعوائد
- **إمكانية الوصول**: عائق منخفض للدخول للاستثمار الزراعي
- **مدفوع بالتكنولوجيا**: منصة حديثة مع نهج محمول أولاً
- **دعم مادي**: العمليات الزراعية الحقيقية توفر أمان الأصول
- **عوائد قابلة للتطوير**: إمكانية النمو من خلال خيارات إعادة الاستثمار

## التطوير

### هيكل Monorepo
```
newyolk/
├── apps/
│   ├── crm/           # نظام CRM الداخلي
│   │   ├── src/
│   │   ├── components/
│   │   └── package.json
│   ├── marketing/     # موقع العملاء
│   │   ├── src/
│   │   ├── components/
│   │   └── package.json
│   ├── expo/          # التطبيق المحمول
│   │   ├── src/
│   │   ├── app.json
│   │   └── package.json
│   └── docs/          # موقع التوثيق
│       ├── pages/
│       ├── components/
│       └── package.json
├── packages/
│   ├── ui/            # مكونات UI المشتركة
│   ├── config/        # التكوينات المشتركة
│   ├── types/         # تعريفات نوع TypeScript
│   └── utils/         # المرافق المشتركة
├── package.json       # package.json الجذر
└── pnpm-workspace.yaml
```

### أوامر Workspace
```bash
# تثبيت التبعيات لجميع workspaces
pnpm install

# تشغيل الأمر في workspace محدد
pnpm --filter crm dev
pnpm --filter marketing build
pnpm --filter expo start

# تشغيل الأمر في جميع workspaces
pnpm -r build
pnpm -r test

# إضافة تبعية إلى workspace محدد
pnpm --filter crm add lodash
pnpm --filter expo add react-native-camera
```

### الحزم المشتركة
- **`packages/ui`**: مكونات UI قابلة لإعادة الاستخدام عبر جميع التطبيقات
- **`packages/config`**: ملفات التكوين المشتركة (ESLint، Tailwind، إلخ.)
- **`packages/types`**: تعريفات وواجهات TypeScript
- **`packages/utils`**: وظائف المرافق الشائعة والمساعدات
